这份手册怎么用

一共三层，从粗到细。前两层在脑子里搭骨架，第三层才是动手。

核心目标：跑通 Day3 Agent、启动 Day4 服务、用 curl 模拟设备发三条消息。有真机的同学再做进阶路线。

关于本手册：以下讲解基于课程包提供的示例主题——校园 AI 助手（整理课程资料、生成摘要/待办/复习卡）。但你的 Agent 不必照抄这个主题。你可以选择任何方向：个人知识库、游戏攻略引擎、学习计划生成、创意写作助手……Agent 的核心骨架（文件读取、RAG 检索、LLM 调用、工具注册、trace 记录）是不变的。你只需要换掉 workspace 里的素材和 prompt 即可。本手册的作用是让你先看懂一条完整链路是怎么搭的，然后你拿这套骨架去实现自己的主题。

一句话：人发消息 → HTTP 请求打到电脑上的 Day4 服务 → Day4 调 Day3 Agent → Agent 读文件、检索知识、调模型、写结果 → 结果通过 HTTP 返回。

Day3 和 Day4 各自负责什么

Day3 — 桌面 Agent
Agent 的大脑。负责读文件、建知识库、RAG 检索、调 LLM、写结果、留 trace。它本身是一个可以双击运行的 Python 脚本，不依赖网络服务。

Day4 — 设备网关
Agent 的耳朵和嘴巴。在电脑上开一个 HTTP 服务（端口 8000），等人来请求。收到请求后调 Day3 Agent 干活，再把结果返回给请求方。

Day3 是"能干活的程序"，Day4 是"让它能被远程触发的壳"。没有 Day4，Agent 只能你手动双击跑一次。有了 Day4，外面发一条消息它就跑一次。

为什么要搭两层？

这是一种常见工程模式——关注点分离。Day3 只管"怎么干活"，不管"谁来叫我干活"。Day4 只管"接客"，不管"怎么干"。好处：

  • 改 Agent 逻辑不影响 HTTP 接口，改接口不影响 Agent 逻辑。
  • 没有真机时，可以双击 Day3 直接验证 Agent 是否正常。
  • 以后想加新的触发方式（网页按钮、定时任务、语音入口），只需要改 Day4，Day3 不用动。

S3 设备在链路里的真实角色

本课程中，人机交互的实际入口是微信。你在微信里发消息给 ClawBot，ClawBot 通过互联网转给 S3 设备，S3 再通过局域网向你的电脑发起 HTTP POST 请求。S3 在这里扮演的是消息中转+结果展示——它收到微信消息后转发给 Day4，收到 Day4 回复后显示在屏幕上。S3 本身不提供文本输入能力，真正的"人说话"发生在微信端。没有真机的同学，用本机的 curl 直接向 Day4 发请求，效果完全等价。

B. Day4：启动服务 + 模拟设备

2. 启动服务

终端会打印两行关键信息：

Local:       http://127.0.0.1:8000/dashboard

这个终端不要关。关了服务就停了。

怎么看自己的局域网 IP

Day4 服务启动时，终端已经打印了你的 IP（Device Base URL 那行）。如果没看到或不确定，手动查：

Mac：打开 系统设置 → 网络 → Wi-Fi，点当前连接旁边的"详细信息"，IP 地址写在里面。或者终端跑：
ipconfig getifaddr en0（Wi-Fi）或 ipconfig getifaddr en1

Windows：打开终端（CMD），输入 ipconfig，找到"无线局域网适配器 Wi-Fi"下面的 IPv4 地址。

注意：不一定是 192.168 开头。也可能是 10.x.x.x、172.x.x.x，取决于路由器。无论哪种，只要 S3 和你的电脑在同一个 Wi-Fi 下，就用这个 IP。

防火墙：必须关

S3 设备通过局域网访问你的电脑，如果防火墙开着，外部连接会被拦截。在启动服务之前，确认防火墙已关闭：

Mac：系统设置 → 网络 → 防火墙 → 关闭。如果系统策略不允许完全关闭，至少确认没有勾选"阻止所有传入连接"。

Windows：控制面板 → Windows Defender 防火墙 → 启用或关闭 Windows Defender 防火墙 → 专用网络和公用网络都选"关闭"。

课堂演示期间临时关闭是安全的。日常使用时记得重新打开。

# 第三条：要一道复习题
curl -X POST http://127.0.0.1:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer classroom-demo-key" \
  -d '{"model":"pocket-campus-agent","messages":[{"role":"user","content":"给我一道复习题"}]}'

curl 命令逐行解释

curl 不是重点，重点是"发 HTTP 请求"。curl 只是帮你发的工具。每一行拆开看：

片段含义
curl命令行发 HTTP 请求的工具
-X POST用 POST 方式（发送数据），不是 GET（读取数据）
http://127.0.0.1:8000/...目标地址：本机、8000 端口、/v1/chat/completions 路径
-H "Content-Type: ..."告诉服务器：我发的正文是 JSON 格式
-H "Authorization: ..."API Key 验证。课堂用 classroom-demo-key，不强校验
-d '{...}'请求正文（body）。花括号里是发给 Agent 的 JSON
"model"告诉 Day4 服务用哪个模型处理
"messages"对话消息数组。每一条有 role（谁说的）和 content（说了什么）
\ 换行为了让命令可读而断开，终端会当成一行执行

D. 进阶：M5Stack CoreS3 真机路线

进阶 没有真机也能完成核心学习目标。以下是有设备的同学的路子。

  1. 用 M5Burner 选 CORES3，烧录 ESP-Claw 固件。
  2. 重启 S3，连它的 Wi-Fi 热点（esp-claw-xxx）。
  3. 浏览器打开 192.168.4.1，配 Wi-Fi 和 LLM 设置：
    Backend Type: openai_compatible
    Config File: qwen_compatible（不要改！改过会导致 S3 重启异常）
    Base URL: http://你的局域网IP:8000/v1
    Model: pocket-campus-agent
    API Key: classroom-demo-key
    Auth Type: bearer
  4. 保存后，S3 连上 Wi-Fi，生成微信绑定二维码。扫码后微信里发消息就能触发你的 Day4 服务。

Config File 保持 qwen_compatible 的原因：ESP-Claw 内部按这个配置名加载请求格式。改 openai_compatible 会让 S3 重启异常，这是真机验证的结论。
